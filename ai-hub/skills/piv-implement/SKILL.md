# PIV Implement Skill
Executes the approved plan strictly. This is Phase 2 of the PIV loop.

## Workflow
1. **Consume Artifact**: You MUST be provided with the path to a plan artifact generated by `piv-plan` (e.g., `ai-hub/plans/P-XXXX.md`). Use the `view_file` tool to read this file fully before starting any work.
2. **Strict Execution**: Implement the steps exactly as outlined in the plan. Modify the necessary code files. 
   - **DO NOT** introduce undocumented changes, scope creep, or refactors outside the plan.
   - If you encounter an unexpected issue that makes the plan unviable, stop and ask the user for permission to adjust the plan.
3. **Local Checks**: Perform basic sanity checks on your changes.
4. **Handoff for Validation**: Once the implementation matches the plan, you MUST explicitly enforce running `python ai-hub/gates/validate.py` and invoke the `piv-validate` skill (or instruct the user/subagent to do so). Do not claim the task is done until the validation gates are strictly enforced by the `piv-validate` phase.
